Classes vs Ids in Ruby

In css, we use ID’s and classes to addresses elements. ID’s and classes, by themselves, don’t have any styling information to them but we can apply styling by targeting them using css. confused? ok let’s define what they means.


ID’s are unique. meaning that an element can only have one ID, where as, classes are not unique. you can apply some class on multiple elements or multiple classes on a same element. when you have one element that requires a certain kind of styling, then ID’s are great to use but when you have a lot of elements that should receive the same styling, classes are better in use.


I read this example on internet when I was browsing to understand them better. take a laptop in store for example, they have barcode and serial number. The serial number is a number which is unique to every laptop like ID’s but the barcode is something that the store puts on the package to identify a certain group of laptops for the purpose of identifying price, color, product type etc. same barcode is used for many laptops of same type like classes.


To summarize, informations that are reusable should be kept in classes and the ones that are unique should be in an ID.
