The Map Method

01/29/2016

The map method is one of the important enumerable or collection operations available in Ruby. What ever enumerable it starts with, map always returns an array. The returned array is always the same size as the original enumerable. Its elements consist of the accumulated result of calling the code block on each element in the original object in turn.

for example:

array =[1,2,3,4] .map {|num| num * 2 } #=> [2,4,6,8]
array #=> [1,2,3,4]

The new array is the same size as the original array, and each of its elements corresponds to the element in the same position in the original array. but each element has been run through the block.

In the above example, even if the result prints out [2,4,6,8], when we call the array it will print out the original value which is [1,2,3,4] . If we want to make the change permanent, we use map!

array =[1,2,3,4] .map! {|num| num * 2 } #=> [2,4,6,8]
array #=> [2,4,6,8]

map returns a new object : a mapping of the original object to a new object. it maintains an accumulator array of the results from the block. lets see an example

array = [1,2,3,4]
result = array.map {|i| puts i * 2 } #=> [nil, nil, nil, nil]

even though it prints out the value in the console, the result of the operation, the mapping it self is nil. the return value of puts is always nil so that every call to this particular block will return nil. so be careful on how to use map with block evaluation.
