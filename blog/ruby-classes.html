Best practices of ruby classes

02/05/2016

Ruby is an object-oriented programming language, it revolves around the process of defining and manipulating objects. However, sometimes we may want to categorize objects to be able to easily identify them. to make an object in ruby, first we need to define the characteristics of a class, then create an instance. A category of objects like "cat" is called a class, and some specific object belongings to a class is called an instance of that class.

To illustrate the process, let's first define a simple Cat class. So what are the attributes that cats have ? lets just say cats have name. We can set these class attributes as instance variables, denoted by a beginning @ symbol.

class Cat def initialize(name) @name = name end def talk ” #{@name} is saying Meow Meow” end end

Instance variables are bound to the class and allow us to access it from any location inside the class. Instance methods are also bound to the class and they are instances of the class like def talk method in the example.

All cats will fall under this class, allowing them to share the same methods and characteristics. A class definition is a block of code between the keywords class, and end . A def inside this block begins the definition of a method of the class , which corresponds to some specific behavior for objects of that class.

now that we have defined our Cat class, we can use it to make a cat.

kitten = Cat.new (“Kitty”)

kitten is a name we have given to the new instance of the class Cat and Kitty is the name of our cat. The new method of any class makes a new instance. Because kitten is a Cat, it has whatever properties we decided a Cat should have. Since we only have one method on our class definition, Kitten can only do one thing which is talk

Kitten.talk #=> Kitty is saying Meow Meow

Making a new instance of a class is sometimes called instantiating that class. We need to have a Cat before we can experience the pleasure of its conversation; we can't simply ask the Cat class to say meow for us. It makes no more sense than trying to eat the concept of a sandwich with out having one.

We can use classes to represent any collection of things like cars, dogs, cats, and so on. Just consider what characteristics and behaviors the objects share. The instance variables will be the object's attributes while the methods represent the object's behavior.
