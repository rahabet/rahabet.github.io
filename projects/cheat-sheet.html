SQL CheatSheet

Name Description
SQL It is how we write to and retrieve data from relational database management system
RDBMS It is a way that data is presented to the end user in the form of tables, columns and rows.
Three value logic Unlike other programming languages,SQL has a three value logic which are True/False and unknown(NULL)
Primary Key It is a column or set of columns that are guaranteed to be unique in a given row.
Foreign Key It is a referential constraint between two tables that informs data integrity.
Result Set The results returned from a SELECT statement.
Predicate An expression that evaluates to True, False or Unknown. We use predicates on WHERE, HAVING and JOINS.
SELECT *
FROM table_name 		This means that return all columns in the table_name at runtime.
SELECT *
FROM table_name
WHERE column_name(=, > )num/string 		the where clause must decide if something is true or false. A row with a non-null value is either true or false for each predicate. It determines which rows to include in the result set.
IN and NOT IN allows you to build lists (Values) : Lets say if asked to find a customer from a Customers table with CustomerId 1,3,4,5 we write:
SELECT *
FROM Customers
WHERE CustomerId IN (1,3,4,5)
BETWEEN allows you to test ranges:
SELECT *
FROM Customers
WHERE CustomerId BETWEEN 10 AND 20
LIKE allows for pattern matching:lets say we want to get all names where the first letter starts by c.we write
SELECT *
FROM Customers
WHERE name LIKE “c%”
DISTINCT SQL does allow duplicates in the result set but the use of DISTINCT eliminate duplicate “rows” from result set but only if the whole row is a duplicate.
TOP returns the top “n” rows and its result can be affected by the ORDER BY. TOP get processed after the query is ordered by something we want.
SELECT TOP(3)
